2. Contextualização (A "Dor" do Problema)
    • Descrição do Cenário: A unidade escolar enfrenta dificuldades no controle de ativos de alto valor, como projetores, notebooks e televisores. Atualmente, o registro é manual, facilitando extravios e erros de inventário. É necessária uma solução automatizada para que professores e colaboradores utilizem seus cartões RFID para registrar retiradas e devoluções em tempo real.
    • Problema: Falta de registro eficiente, filas, perda de equipamentos, falta de informação de quem está com os equipamentos.
    • Solução Proposta: Desenvolver um sistema para registro e acompanhamento dos empréstimos, e também a implementação do controle usando Cartões e TAGs RFID.
3. Missão do Sistema
    • Objetivo Geral: Uma frase que defina o propósito central do software. 
            ▪ Deve ser escrita de forma colaborativa com os interessados.
4. Atores do Sistema
    • Identificação: Listar quem são os usuários e as fronteiras do sistema.
    • Exemplos: Professor, Secretária, Administrador do Sistema.
5. Escopo e Fronteiras
    • O que o sistema fará: Lista das principais funcionalidades que definem o limite do projeto integrador.
    • O que o sistema não fará: Definição clara do que está fora da responsabilidade atual do desenvolvimento.
6. Requisitos Funcionais (RF)
    • Descrição: Funções principais que o sistema deve executar para atender às necessidades do usuário.
    • Exemplo: O sistema deve ler etiquetas RFID para registrar o empréstimo.
7. Requisitos Não Funcionais (RNF)
    • Critérios de Qualidade: Atributos relacionados ao desempenho, segurança, usabilidade e robustez.
    • Exemplo: O tempo de leitura da tag RFID deve ser inferior a 2 segundos.
8. Regras de Negócio (RN)
    • Lógica do Processo: Diferenciar os requisitos das regras que regem o negócio.
    • Exemplo: “Para pegar um kit, o professor deve obrigatoriamente validar seu cartão de identificação” (Include).
9. Critérios de Sucesso
    • Resultados Esperados: O que define que o projeto foi bem-sucedido (Ex: redução de 90% na perda de equipamentos)
